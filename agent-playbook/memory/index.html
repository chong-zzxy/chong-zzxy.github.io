记忆系统设计要点：准入、落盘、注入、遗忘与团队同步。

01记忆系统

本章目标：建立“采集 → 准入 → 写入 → 注入 → 清理 → 同步”闭环，让记忆稳定、可追溯、可纠偏。

01记忆系统 02主循环 03工具编排 04多 Agent 05可观测迁移

1. 为什么记忆系统经常越做越差

常见退化路径：

  1. 把临时信息写成长期事实。
  2. 只写不清理，历史噪声累计。
  3. 检索命中很多，但注入质量很低。
  4. 团队共享时冲突覆盖，事实漂移。

所以重点不是“有没有记忆”，而是“记忆生命周期是否受控”。

2. 设计逻辑：六段闭环

  1. 候选提取：从对话与工具结果中提取候选记忆。
  2. 准入判定：过滤临时、可推导、敏感、冲突项。
  3. 写入落盘：区分本地记忆与团队记忆。
  4. 检索注入：按当前任务相关度注入，不做全量塞入。
  5. 预算治理：超预算时先去重，再压缩，再降级。
  6. 遗忘纠偏：过期、替换、删除与冲突回滚。

3. 一个最小可运行的准入器（Python）

4. 注入不是越多越好

建议顺序：

  1. 当前任务必需信息。
  2. 最近确认的人名/角色映射。
  3. 高相关历史决策。
  4. 低优先级背景信息（预算允许才注入）。

这样能避免“记忆很多，但答案更差”的反直觉问题。

5. 风险清单

  1. 污染风险：错误人名映射被长期记住。
  2. 陈旧风险：旧事实覆盖新事实。
  3. 膨胀风险：上下文被历史记忆拖慢。
  4. 同步风险：团队写入冲突导致事实分叉。

如果你只做写入，不做纠偏，系统会慢慢失真。
